# TraceMark 记忆文档

## 分析原则（2026-05-04 确立）

1. **从小到大**：从最小、最简单的案例开始分析
2. **依赖已有规律**：分析大案例时，依靠前面已发现的规律
3. **谨慎推断**：不符合已知模式的现象，不轻易下结论
4. **追求详细**：深入细致地分析，而非泛泛而谈

## PPS 到 Y 的转换规则（2026-05-04 更新）

### 正式规则
1. **基础**：`pps_to_y(0) = y(1)`
2. **后继展开**：`pps_to_y(A, 0) = y(pps_to_y(A), 1)`
3. **极限展开**：`pps_to_y(A) = y(B)`，满足区间关系：
   `pps(A[n]) < pps(y_to_pps(B[m])) < pps(A[n+1])`

### 说明
- 规则3：PPS序列A的展开与Y序列B的展开是**交错区间**关系
- 只在极限类型序列上使用规则3
- 断言失败的序列不适用

## 观察与发现

### PPS 系统
- 简单序列 `0 1` 展开为无限重复的 0
- 序列 `0 1 2` 展开为 0 后跟无限重复的 1
- 序列 `0 1 2 2` 展开为交替模式：0,1,2,1,4,1,6,1,8,...
- 序列 `0 1 0` 提前终止
- 序列 `0 1 0 1` 产生复杂的交替模式
- 序列 `0 1 0 0 3` **快速收敛到0**
- 序列 `0 1 2 3` 展开为 0,1,2 后跟无限重复的 2

### Y 系统
- 简单序列 `1 2` 展开为无限重复的 1
- 序列 `1 2 3` 展开为 1,2 后跟无限重复的 2
- 序列 `1 2 3 3` 展开为交替模式：1,2,3,2,3,2,3,...
- 序列 `1 2 1 1 4` **指数增长**：1,2,1,1,3,9,27,81,243,...

### PPS vs Y 关键差异
| 特性 | PPS | Y |
| :--- | :--- | :--- |
| 起始元素 | 0 | 1 |
| 空序列行为 | 无输出 | 断言失败 |
| `0 1 0 0 3` vs `1 2 1 1 4` | 收敛到0 | 指数增长 |
